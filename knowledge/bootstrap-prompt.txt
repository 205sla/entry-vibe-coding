### B) `.ent` 생성 툴킷 (`tools/make-ent.mjs`)
범용 generator로 설계.
- 입력: 플레인 JS 객체 스펙
  ```js
  {
    name: "Tetris",
    objects: [{ name, pictures:[{path}], sounds:[{path}], script:[[block,...]],
                entity:{x,y,regX,regY,scaleX,scaleY,direction} }],
    variables: [{name, value}],
    lists: [{name, array}],
    scenes: [{name}]
  }
  ```
- 처리:
  1. `entryjs/src/class/project.js` 의 `Entry.getStartProject()` 기본 shape에서 출발.
  2. 각 picture.path가 로컬 파일이면 32자 해시로 이름 붙여 `temp/aa/bb/image/` 아래 번들. SVG → sharp로 PNG 래스터 + 96px 썸네일.
  3. `script` 배열을 `JSON.stringify` → `object.script`.
  4. tar (ustar, dir 0755, file 0644) → gzip memLevel 6 → Buffer 반환.
- 레퍼런스: `MYentry/.claude/skills/code205-problem/make-ent.mjs` 의 tar 파싱·빌드 로직. 같은 함수 이식 OK.

### C) 블록 type 레지스트리 (`tools/build-block-registry.mjs`)
- 빌드 스크립트가 `entryjs/src/playground/blocks/block_*.js`를 AST로 스캔.
- 각 모듈의 `getBlocks()` 반환값에서 `{type, category, params, statements, paramsKeyMap}`를 추출.
- `tools/block-registry.json`으로 저장.
- `make-ent.mjs`와 smoke 테스트가 이 레지스트리로 "존재하지 않는 블록 type", "params 개수 불일치" 같은 오류를 사전 차단.

### D) 자동 테스트 파이프라인

**1층: Node 스모크 (`tests/smoke.test.js`, `node --test`)**
`.ent` 파일 1개를 받아:
- gunzip + tar 파싱 성공
- `temp/project.json` 존재, JSON 파싱 성공
- 필수 top-level 키 (`objects`, `scenes`, `variables`) 존재
- 모든 `objects[*].script`가 문자열 & parse 시 2D 배열
- 모든 block `type`이 `block-registry.json`에 존재
- 모든 `objects[*].sprite.pictures[*].fileurl` 경로가 tar에 실재
- 모든 `selectedPictureId`가 해당 object의 pictures[*].id에 실재
- 모든 `objects[*].scene`이 `scenes[*].id`와 매칭
- 실패 시 어느 키·어느 블록이 어긋났는지 구체적으로 보고

**2층: Playwright 헤드리스 (`tests/e2e.spec.js`)**
- 테스트 전 `npm start`가 백그라운드 돌고 있다고 가정 (playwright.config에 webServer 설정).
- 헤드리스 크로미움 → `http://localhost:3000/editor.html`
- 페이지가 로드되면 `window.Entry`가 존재할 때까지 대기.
- fixture `.ent`를 fetch로 ArrayBuffer 로드 → FormData → `/api/load` → 응답 project JSON을 `Entry.loadProject(data)` 로 로드.
- 3초 이내 `Entry.container.objects.length > 0` 확인.
- `Entry.engine.toggleRun()` → 1초 후 `Entry.container.getAllObjects().some(o => o._warningBlock)` false여야 함.
- `window.onerror` 후킹한 배열이 빈 배열이어야 함.
- `Entry.engine.toggleStop()` 호출 후 종료.

루트 `npm run verify` = smoke → e2e 순차 실행.

### E) 레퍼런스 문서 `docs/ent-format.md`
외부 LLM이 읽고 직접 `.ent`를 만들 수 있을 만큼 구체적으로 작성.
1. tar.gz 포맷 (헤더 mode 000755/000644, ustar 마커, gzip memLevel 6)
2. `project.json` 전체 스키마 (각 키의 타입·기본값·필수 여부) — entryjs `src/class/project.js` 참조
3. Object / Entity / Picture / Sound 필드 레퍼런스
4. Variable / List / Timer / Answer 필드 + 화면 좌표 관례
5. Block 스크립트 JSON 포맷 (thread 2D 배열, params·statements 중첩)
6. 대표 블록 15종 예시 (게임 제작 관점):
   - events: `when_run_button_click`, `when_some_key_pressed`, `when_message_cast`
   - flow: `repeat_basic`, `repeat_inf`, `wait_second`, `wait_until_true`, `stop_repeat`
   - judgement: `if_basic`, `if_else_basic`, `boolean_and_or`, `boolean_comparison`
   - moving: `move_direction`, `move_xy_time`, `locate_xy`, `bounce_wall`
   - looks: `dialog`, `change_to_next_shape`, `change_to_some_shape`, `set_effect`, `set_scale`
   - variable: `set_variable`, `change_variable`, `get_variable`
   - list: `add_value_to_list`, `remove_value_from_list`, `value_from_list`
   - sound: `sound_something_with_block`, `sound_something_wait_with_block`
   - func: `function_general`, `function_param_string`
7. 자산 해시 규칙 (32자, aa/bb 샤딩, SVG+PNG 쌍)
8. 로드 실패 트러블슈팅
   - 블록 빈칸 → `Entry.block[type]` 미등록, 블록 type 오타
   - 이미지 회색 → fileurl 경로 오류 또는 tar mode 오류
   - 변수 안 보임 → variableType 오타, visible:false
   - 스크립트 안 붙음 → script가 문자열이 아니거나 평면 배열
   - loadProject 즉시 에러 → scenes[0].id와 objects[*].scene 불일치
   - 실행 시 경고 빨간 블록 → params 개수/타입 불일치 (레지스트리로 사전 감지 가능)
9. 레퍼런스 파일 경로 목록 (entryjs 소스 파일:라인)
